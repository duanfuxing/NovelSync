# 客户端数据库表结构设计 (Client Local SQLite 3)
为了保障端云数据结构的一致性与解耦同步，本客户端 SQLite 表结构设计严格对齐服务端 (`novel_baidu_subscription_` 前缀系列) 的字段规范。客户端表统一采用 `client_` 作为前缀，确保字段类型与服务端 MySQL 高度对应。
使用 **SQLModel** 作为 ORM 层映射此本地 SQLite 3 数据库。运行前必须开启 `PRAGMA journal_mode=WAL;` 以支持 FastAPI 与后台守护多进程池的高效并发读写。
---
### 1. 全局系统与用户配置表 (`client_config`)
用于保存持久化的登录身份（Token + UID）与基础路径配置，对应设备上下文。
```sql
CREATE TABLE client_config (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    uid TEXT UNIQUE, -- 服务器分配的用户UID
    token TEXT NOT NULL, -- 身份验证JWT令牌
    client_id TEXT NOT NULL, -- 本机硬件哈希唯一标识 (Device MAC)
    watch_path TEXT -- 用户设定的本地小说原文目录监控路径
);
```
### 2. 本地百家号身份池缓存表 (`client_bjh_cookies`)
用于存放并调度从远端 `/sync/bjh/cookies` 拉取同步回来的可用账号列表，映射服务端 `novel_baidu_subscription_cookies`。
```sql
CREATE TABLE client_bjh_cookies (
    bjh_id TEXT PRIMARY KEY, -- 百家号唯一标识ID
    bjh_name TEXT NOT NULL, -- 百家号名称
    bjh_avatar TEXT, -- 头像URL
    cookie_str TEXT NOT NULL, -- Cookie 凭证完整数据
    status INTEGER DEFAULT 1, -- 账户可用状态 (被服务端冻结时设为0)
    last_used DATETIME -- 上次该账号被调度执行抓取的时间(用于多推文号均匀轮询)
);
```

### 3. 文章元数据同步队列表 (`client_articles`)
作为文章抓取进度的本地暂存队列，对应服务端的 `novel_baidu_subscription_articles` 表，缓存后按批次异步发送上报。
```sql
CREATE TABLE client_articles (
    article_id TEXT PRIMARY KEY, -- 发布的百家号平台原子文章实体 ID
    nid TEXT, -- 百家号分配的小说或合集核心关联ID
    title TEXT, -- 文章/章节名称
    abstract TEXT, -- 文章摘要
    feed_id TEXT, -- 信息流分配ID
    app_id TEXT, -- 百家号应用ID
    story_type TEXT, -- 故事类型 (如 subscribe)
    url TEXT,  -- 公网前台可浏览链接
    vertical_cover TEXT, -- 文章竖版封面图 URL
    status TEXT, -- 文章总体状态 (如 publish)
    type TEXT, -- 动态内容类型 (如 news)
    is_published INTEGER DEFAULT 0, -- 是否已发布 (0/1)
    is_pay_subscribe INTEGER DEFAULT 0, -- 是否挂载了付费专栏/合集的文章 (0/1)
    publish_time DATETIME, -- 源平台原始分发的发表时间
    sync_status INTEGER DEFAULT 0, -- 双向通讯上报状态 (0:未同步待上报, 1:已同步完成)
    last_update DATETIME -- 本地最后一次抓取/更新时刻
);
```
### 4. 订单与营收表现同步队列表 (`client_orders`)
【新增】作为全局小说跑单业绩的本地汇总缓存，对应服务端的 `novel_baidu_subscription_orders` 表，记录被订阅专栏/跑单书籍的全局充值转化规模。
```sql
CREATE TABLE client_orders (
    nid TEXT PRIMARY KEY, -- 小说标识ID (作为归属小说级维度的汇总键)
    title TEXT, -- 小说或合集/专栏标题名称
    status TEXT, -- 专栏总体状态或该小说标识状态 (如 publish)
    order_amount INTEGER DEFAULT 0, -- 该专栏产生的累计充值流水订单数或总交易额度
    read_amount INTEGER DEFAULT 0, -- 合集/专栏层面的总阅读量
    rec_count INTEGER DEFAULT 0, -- 官方渠道推荐次数/推荐量指数
    comment_amount INTEGER DEFAULT 0, -- 专栏维度的总体评论数
    like_amount INTEGER DEFAULT 0, -- 专栏维度的总体点赞数
    collection_amount INTEGER DEFAULT 0, -- 用户收藏书架的总规模数
    share_amount INTEGER DEFAULT 0, -- 专栏累计被分享次数
    is_hot INTEGER DEFAULT 0, -- 平台热门跑量作品标记 (如 0 或 1)
    is_pay_subscribe INTEGER DEFAULT 1, -- 是否启用了付费订阅跑单专栏 (0/1)
    sync_status INTEGER DEFAULT 0, -- 双向通讯上报状态 (0:未同步待上报, 1:已同步完成)
    last_update DATETIME -- 本地最后一次抓取宏观指标的更新时刻
);
```
### 5. 原稿提取与文件同步任务表 (`client_books_sync_tasks`)
重构自原旧版 file_sync_tasks。作为本地文档提取与上传状态的断点记录表，最终映射投递入服务端的 `novel_baidu_subscription_books` 主表内。
```sql
CREATE TABLE client_books_sync_tasks (
    file_hash TEXT PRIMARY KEY, -- 原始本地文件的 MD5 哈希(鉴别重复依据)
    nid TEXT, -- 隶属的小说核心关联ID
    novel_id TEXT, -- 文章实体/平台小说原始业务 ID
    title TEXT, -- 小说大名称或对应章节名
    local_path TEXT, -- 本地文件所在的绝对路径
    file_size INTEGER, -- 文件大小 (bytes)
    word_count INTEGER DEFAULT 0, -- 从 doc/docx 解析统计出的纯文字字数
    sync_status TEXT, -- 当前阶段 (PENDING, EXTRACTING-提取中, UPLOADING-上云中, COMPLETED-成功, FAILED-失败)
    error_msg TEXT, -- 失败中断时的异常原因描述保存
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP -- 扫描感知到新文件触发入列的时间
);
```
