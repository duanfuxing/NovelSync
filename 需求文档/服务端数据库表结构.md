# 百家号小说数据同步 - Server 端表结构设计

该文档基于 `api_spec.md` 定义的接口需求，为远端业务后台设计关系型数据库表结构 (MySQL 8.0+)。

> **用户体系说明**: 本模块复用妙笔系统现有的 `users` 表，所有业务表通过 `user_id` 关联 `users.id`，不再独立维护用户表和登录日志表。

## 通用设计约定
* **字符集与排序规则**: `utf8mb4` / `utf8mb4_unicode_ci`。
* **引擎选择**: `InnoDB`。
* **金额字段**: 统一定义为 `DECIMAL(10,2)`，以"元"作为基础单位。
* **公共审计字段**: 表中常规保留 `created_at` (创建时间) 与 `updated_at` (修改时间)。
* **用户关联**: 所有表使用 `user_id` (BIGINT UNSIGNED) 关联妙笔系统 `users.id`。

## 1. 配置与控制模块

### 1.1 原始 Cookie 信息表 (`novel_baidu_subscription_cookies`)
统一管理系统录入的有效百家号 Cookie 凭证及百家号账号基础元信息。

```sql
CREATE TABLE `novel_baidu_subscription_cookies` (
  `id` BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '物理主键标识',
  `cookie_str` TEXT NOT NULL COMMENT '百家号平台验证有效的 Cookie 原始字符串内容',
  `bjh_avatar` VARCHAR(1024) DEFAULT NULL COMMENT '百家号头像',
  `bjh_name` VARCHAR(255) DEFAULT NULL COMMENT '百家号名称',
  `bjh_id` VARCHAR(64) NOT NULL COMMENT '百家号ID (标识)',
  `status` TINYINT DEFAULT 1 COMMENT 'Cookie 状态 (1: 正常可用, 0: 已失效)',
  `created_at` TIMESTAMP NULL DEFAULT CURRENT_TIMESTAMP COMMENT '录入时间',
  `updated_at` TIMESTAMP NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '最后更新或校验时间',
  UNIQUE KEY `uk_bjh_id` (`bjh_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='百家号原始Cookie信息表';
```

### 1.2 用户与 Cookie 关联表 (`novel_baidu_subscription_user_cookie_relation`)
将妙笔系统用户与授权其使用的百家号 Cookie 建立多对多的关联映射。

```sql
CREATE TABLE `novel_baidu_subscription_user_cookie_relation` (
  `id` BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '物理主键',
  `user_id` BIGINT UNSIGNED NOT NULL COMMENT '妙笔系统用户 ID (关联 users.id)',
  `cookie_id` BIGINT NOT NULL COMMENT '关联的 Cookies 表的主键 ID',
 `created_at` TIMESTAMP NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` TIMESTAMP NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX `idx_user_id` (`user_id`),
  INDEX `idx_cookie_id` (`cookie_id`),
  UNIQUE KEY `uk_user_cookie` (`user_id`, `cookie_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户与Cookie授权关联表';
```

### 1.3 客户端设备状态机/会话表 (`novel_baidu_subscription_client_sessions`)
承载 WS 心跳和健康检测上报，监控各个客户端健康情况与负载状态。

```sql
CREATE TABLE `novel_baidu_subscription_client_sessions` (
  `id` BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '物理主键标识',
  `user_id` BIGINT UNSIGNED NOT NULL COMMENT '妙笔系统用户 ID (关联 users.id)',
  `client_id` VARCHAR(64) NOT NULL COMMENT '客户端设备硬件唯一MAC码识别位',
  `is_online` TINYINT DEFAULT 0 COMMENT '网络 WS 连接是否在线 (1: 在线, 0: 离线)',
  `is_cookie_valid` TINYINT DEFAULT 1 COMMENT '使用的 Cookie 是否实测有效',
  `running_tasks` INT DEFAULT 0 COMMENT '当前挂起的并发执行任务数',
  `cpu_usage` DECIMAL(5,2) DEFAULT NULL COMMENT '硬件负载 - CPU 占用百分率',
  `mem_usage_mb` INT DEFAULT NULL COMMENT '硬件负载 - 内存占用量 (MB)',
  `last_heartbeat` TIMESTAMP NULL DEFAULT NULL COMMENT '最后一条安全心跳确认时间',
  `last_sync_time` TIMESTAMP NULL DEFAULT NULL COMMENT '最近一次真实的数据执行时间',
  `created_at` TIMESTAMP NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` TIMESTAMP NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  UNIQUE KEY `uk_client_id` (`client_id`),
  INDEX `idx_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='客户端设备状态机会话表';
```

## 2. 业务数据接收模块

### 2.1 小说内容原文表 (`novel_baidu_subscription_books`)
收录推报的实体小说结构信息，支持容灾分析或云端接管处理。

```sql
CREATE TABLE `novel_baidu_subscription_books` (
  `id` BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
  `user_id` BIGINT UNSIGNED NOT NULL COMMENT '妙笔系统用户 ID (关联 users.id)',
  `nid` VARCHAR(64) DEFAULT NULL COMMENT '百家号nid',
  `novel_id` VARCHAR(64) DEFAULT NULL COMMENT '百家号文章ID',
  `title` VARCHAR(255) DEFAULT NULL COMMENT '小说标题',
  `content` LONGTEXT COMMENT '小说正文内容',
  `word_count` INT DEFAULT 0 COMMENT '解析统计的文章字数',
  `url` VARCHAR(1024) DEFAULT NULL COMMENT '百家号文章url',
  `vertical_cover` VARCHAR(1024) DEFAULT NULL COMMENT '百家号竖版封面图 URL',
  `status` VARCHAR(32) DEFAULT NULL COMMENT '百家号文章总体状态 (如 publish)',
  `type` VARCHAR(32) DEFAULT NULL COMMENT '百家号动态内容类型 (如 news)',
  `is_published` TINYINT DEFAULT 0 COMMENT '百家号平台状态判定：是否已发布',
  `is_pay_subscribe` TINYINT DEFAULT 0 COMMENT '是否为挂载了付费专栏/合集的文章',
  `read_amount` INT DEFAULT 0 COMMENT '单篇文章阅读量',
  `like_amount` INT DEFAULT 0 COMMENT '单篇文章点赞数',
  `comment_amount` INT DEFAULT 0 COMMENT '单篇文章评论数',
  `profit` DECIMAL(10,2) DEFAULT 0.00 COMMENT '单篇预估收益 / 交易分成',
  `publish_time` TIMESTAMP NULL DEFAULT NULL COMMENT '源平台原始分发的发表时间',
  `created_at` TIMESTAMP NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` TIMESTAMP NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX `idx_user_id` (`user_id`),
  INDEX `idx_nid` (`nid`),
  INDEX `idx_novel_id` (`novel_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='小说内容原文表';
```

### 2.2 百家号小说订阅订单数据表 (`novel_baidu_subscription_orders`)
定期抓取的关于小说/合集整体规模的变现表现及互动指标情况表。

```sql
CREATE TABLE `novel_baidu_subscription_orders` (
  `id` BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
  `user_id` BIGINT UNSIGNED NOT NULL COMMENT '妙笔系统用户 ID (关联 users.id)',
  `nid` VARCHAR(64) NOT NULL COMMENT '百家号nid',
  `novel_id` VARCHAR(64) DEFAULT NULL COMMENT '百家号文章ID',
  `status` VARCHAR(32) DEFAULT NULL COMMENT '专栏总体状态或该小说标识状态 (如 publish)',
  `order_amount` INT DEFAULT 0 COMMENT '该专栏产生的累计充值流水订单数或总交易额度',
  `read_amount` INT DEFAULT 0 COMMENT '合集/专栏层面的总阅读量',
  `rec_count` INT DEFAULT 0 COMMENT '官方渠道推荐次数/推荐量指数',
  `comment_amount` INT DEFAULT 0 COMMENT '专栏维度的总体评论数',
  `like_amount` INT DEFAULT 0 COMMENT '专栏维度的总体点赞数',
  `collection_amount` INT DEFAULT 0 COMMENT '用户收藏书库的总规模数',
  `share_amount` INT DEFAULT 0 COMMENT '专栏累计被分享次数',
  `is_hot` TINYINT DEFAULT 0 COMMENT '平台热门跑量作品标记 (如 0 或 1)',
  `is_pay_subscribe` TINYINT DEFAULT 1 COMMENT '是否启用了付费订阅跑单专栏',
  `created_at` TIMESTAMP NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` TIMESTAMP NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX `idx_user_id` (`user_id`),
  INDEX `idx_nid` (`nid`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='百家号小说订阅订单数据表';
```
