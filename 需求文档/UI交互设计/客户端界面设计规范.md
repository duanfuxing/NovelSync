# SyncMaster 客户端 UI 交互设计规范

客户端操作界面基于 React + TypeScript 配合 Ant Design 框架打造，使用 Tauri 进行桌面端封装构建与跨进程通信调用本地 FastAPI 核心，划分为独立专属目录存放，核心包含以下几个功能控制面板：

## 1. 登录模块 (Login)
* **包含元素**: 手机号输入框、密码输入框、登录触发按钮、记住我选项。
* **交互逻辑**: 
  * 首次打开未带可用 Token 时，直接阻断进入并展示该登录页。
  * 获取本机 MAC 设备特征结合手机密码调用网关 `POST /auth/login`。
  * 登录鉴权通过后记录令牌至 SQLite 配置文件，随后跳转挂载至主框架界面。

## 2. 设置模块 (Local Settings)
* **包含元素**: 本地目录可视化选择器、当前监控路径展示栏、系统环境探针与保存按钮。
* **交互逻辑**:
  * 用户必须通过文件选择器，在本地磁盘上指定一个独立且具备读写权限的文件夹，作为小说原稿件的自动同步目标目录。
  * 该绝对路径会被持久化保存在本地 SQLite 的 `client_config` 表的 `watch_path` 字段中。
  * **主进程启动强校验机制**: 每次软件冷启动时都必须无条件检查。若系统校验发现 `watch_path` 设定不合法、或该文件夹未配置/被外部删毁，必须强制阻隔所有对应的 Worker 后台管线启动，并立刻向 UI 层发送显眼通知，引导用户重新通过设置模块补齐这部分设定后，才允许解甲运转。

## 3. 本地小说列表 (Local Novels)
* **包含元素**: 带有快速响应的数据视图(Data Table) 或卡片流、万能搜索框、表头多维度排序触发器。
* **呈现数据深度融合**: 
  * *基础维度*：展示并监测本地捕获到的小说原文件的基础画像（文件名、对应小说底层源 `novel_id`、解析出的 `word_count` 字数及识别提取进度）。
  * *指标维度*：深度融合后台 `article_sync` 抓取自服务端及挂载号的业务侧核心数据。展示并对齐对应的最新业务字段包括但不限于：`阅读量 (read_amount)`、`点赞数 (like_amount)`、`评论数 (comment_amount)`、`单篇预估变现 (profit)` 以及向云端的 `双向同步状态 (sync_status)` 等。
* **交互体验**:
  * 支持右上方的动态**搜索筛选**，根据文章/章节标题快速定位模糊检索。
  * 列表头部所有的可量化字段必须加装**一键排序**（如：依收益高低排序、按阅读量热度排序、按本地最终更新时序正/倒排），方便运营人员即时盘点。

## 4. 订单列表 (Macro Orders)
* **包含元素**: 全局充值转化流水分页或流式无尽表格、条件检索控件、核心漏斗指标的表头排序器。
* **呈现数据**: 
  * 展示 `order_sync` 进程归整上报的针对小说的宏观订阅变现业务指标：`订单转化量 (order_amount)`、`公域推送流 (rec_count)`、`私人收藏累加 (collection_amount)`、`二次分发/分享规模 (share_amount)` 等大盘转化率。
* **交互体验**:
  * 搜索模块：开放查询通道，支持直接根据 `nid` (合集ID) 或 小说标书名 进行匹配。
  * 排序赋能：同样支持双向箭头排序切分，可随意点击某列（如最高订单转化、最高被推流数），瞬间透视当下的爆款吸金分布。
