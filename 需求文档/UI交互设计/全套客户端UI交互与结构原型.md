## 1. 登录界面 (Login View)

进入客户端时首先显示的鉴权页面。

### 1.1 UI 原型线框图

```text
+-------------------------------------------------------------+
| SyncMaster v2.0                                      [─][□][X]|
+-------------------------------------------------------------+
|                                                             |
|           +-------------------------------------+           |
|           |                                     |           |
|           |        [ Logo ] SyncMaster          |           |
|           |                                     |           |
|           |  手机号码                           |           |
|           |  [ 📱  请输入手机号                ]  |           |
|           |                                     |           |
|           |  密码                               |           |
|           |  [ 🔒  ********************      ]  |           |
|           |                                     |           |
|           |  [√] 保持登录状态                     |           |
|           |                                     |           |
|           |        [      登 录   ]              |           |
|           |                                     |           |
|           +-------------------------------------+           |
|                                                             |
+-------------------------------------------------------------+
```

### 1.2 React (Ant Design) 组件体系
* **`Layout`**: 外层背景容器，设置 Flex 居中。
    * **`Card`**: 登录白色卡片，带圆角和阴影。
        * **`Form`**: 纵向排列内部验证元素并可受控。
            * `Image`/`Icon`: 顶部 Logo。
            * `Input` (手机号): 输入框，配置 `prefix={<PhoneOutlined />}`。
            * `Input.Password` (密码): 密码输入框，原生支持明暗文切换，配置 `prefix={<LockOutlined />}`。
            * `Checkbox`: “保持登录状态”复选框，勾选后交由 Zustand 响应并落库。
            * `Button` (type="primary"): “登录”按钮，点击触发 Axios 请求跨域调用本地被 Tauri 代理的 FastAPI。

## 3. 小说数据列表 (Novel Data Table)

综合展示基础资料、用户阅读点赞交互以及订单收益情况的数据统筹页面。

### 3.1 UI 原型线框图

```text
+-------------------------------------------------------------+
| [ 🔍 书名/作者查询 ]                      [ 导出账单 / 刷新 ]  |
| ----------------------------------------------------------- |
| ID | NID   | Title (书名)     | 阅读/点赞   | 订单/收益      | 状态   |
| ----------------------------------------------------------- |
| 1  | 99201 | 遮天纪元         | 2.1w / 231  | 402单(¥324.5)| 🟢 正常 |
| 2  | 99511 | 赛博修真传       | 54   / 2    | 1单  (¥12.0) | 🟡 提取中 |
| 3  | 88421 | 银河帝国         | 0    / 0    | 0    (¥0.0)  | 🔴 断开 |
+-------------------------------------------------------------+
```

### 3.2 React (Ant Design) 组件体系
* **`Layout.Content`**: 撑开占据右侧全部显示空间。
    * **`Flex`** (顶部操作栏): 横向弹性排列。
        * `Input.Search`: 轻量的查询搜索框，加装 `onSearch` 延迟过滤触发。
        * `Button`: 数据导出或手动刷新按钮。
    * **`Table`** (数据表格):
        * `columns`: 列配置包含 `id`, `nid`, `title`, 阅读/点赞, 订单/收益, 状态，均配置 `sorter` 表头排序。
        * `dataSource`: 填充数组对象数据。
            * [渲染方式]: 利用 `render` 动态拼接返回各列视图。
            * [各项指标]: `Space` 挤压显示横向拉伸拼接，收益字段包裹 `<Typography.Text type="danger">` 高亮。
            * [状态展示]: 使用 `Badge` 状态徽标 组件辅助展现 `text`。

## 4. 百家号列表 (Baijiahao List View)

展示系统分配给当前客户端关联的百家号信息。

### 4.1 UI 原型线框图

```text
+-------------------------------------------------------------+
| [ 🔍 搜索名称... ]                                            |
| ----------------------------------------------------------- |
| ID | 头像          | 百家号名称                                 |
| ----------------------------------------------------------- |
| 1  | [🖼]          | 互联网那点事                                |
| 2  | [🖼]          | 历史百家说                                 |
| 3  | [🖼]          | 情感树洞                                   |
+-------------------------------------------------------------+
```

### 4.2 React (Ant Design) 组件体系
* **`Layout.Content`**: 面板容器。
    * **`Input.Search`**: 顶部搜索栏。
    * **`Table`** (数据表格):
        * `columns`: 列名为 ID, 头像, 名称。
        * `dataSource`: 列表数据源对象。
            * [ID]: `Typography.Text` 的数字序号。
            * [头像]: 使用 `Avatar` 组件结合 `src` 属性展现图片。
            * [名称]: 纯文本渲染即可。

## 5. 设置界面 (Local Settings View)

用于配置本地基础环境与同步源文件夹。

### 5.1 UI 原型线框图

```text
+-------------------------------------------------------------+
|  ⚙  系统设置                                                  |
| ----------------------------------------------------------- |
|  文件同步目录:                                                |
|  📁 /Users/dl/Desktop/novels_repo             [ 修改监听路径 ] |

| ----------------------------------------------------------- |
|                          [ 保 存 并 重 启  ]                  |
+-------------------------------------------------------------+
```

### 5.2 React (Ant Design) 组件体系
* **`Flex`** 设置 `vertical` 属性进行从上向下的常规堆叠。
* **`Button` 触发器**: 通过 Tauri 自带的 API `@tauri-apps/api/dialog.open` 唤醒系统原生文件夹选择界面，而不是直接使用 input。
* **`Typography.Text`**: 展现验证过后从 Tauri Rust 侧传递过来的当前绝对路径目录。
* **`Button`**: 底部保存按钮，做判空校验后记录传给后端持久化。

## 6. 账号下线弹窗 (Account Disabled Modal)

当发生账号在服务器被禁用等事件时，强制全屏打断当前界面的弹窗。

### 6.1 UI 原型线框图

```text
+-------------------------------------------------------------+
|    [ 底 层 所 有 界 面 立 即 被 黑 灰 半 透 明 蒙 版 强 遮 罩 ]     |
|                                                             |
|       +=======================================+             |
|       |                                       |             |
|       |              [ ⚠️ 图标 ]              |             |
|       |           服务连接已被切断            |             |
|       |                                       |             |
|       |   当前账号已被服务端运营后台冻结/禁用。   |             |
|       |                                       |             |
|       |           [  退 出  ]                  |            |
|       |                                       |             |
|       +=======================================+             |
+-------------------------------------------------------------+
```

### 6.2 React (Ant Design) 组件体系
* **`Modal` (模态弹窗组件)**: 核心弹出拦截界面。
    * 设置 `keyboard={false}` 和 `maskClosable={false}` 防止用户通过点按遮罩或ESC键关闭它。
    * 覆盖全屏高亮的警告UI：
        * 内容配置包含警告图标 `Icon`，说明文字以及唯一出口属性按钮 `<Button type="primary" danger>` 引发 Tauri 环境结束应用的系统调用。
